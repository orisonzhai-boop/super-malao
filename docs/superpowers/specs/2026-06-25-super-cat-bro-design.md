# 超级喵拉奥 (Super Cat Bro) — 设计规格

- 日期：2026-06-25
- 状态：已与用户对齐设计，待评审后进入实现计划
- 类型：浏览器端横版平台跳跃游戏（类超级马里奥，单关）

## 1. 概述

一个在浏览器里玩的横版平台跳跃单关游戏。主角是用户提供的手绘"猫耳 + 眼镜"小人，从关卡起点跑跳到终点旗杆，沿途踩小怪、吃金币、躲深坑。

技术上是**自包含的单个 HTML 文件**：原生 JS + Canvas，零依赖、零构建、双击即玩、可离线运行、便于分享。主角的真实手绘帧以 base64 内嵌进该文件。

名字"超级喵拉奥 / Super Cat Bro"为占位，可后续替换。

## 2. 目标与非目标

### 目标
- 单关、能完整通关的平台跳跃体验，体现马里奥核心手感（跑、变高跳、踩怪、吃币、躲坑、到旗杆通关）。
- 主角直接使用用户手绘原画（`assets/hero_sheet.webp`）切帧得到的真实 sprite。
- 涂鸦风主角与彩色像素世界混搭，通过白色描边光晕 + 落地阴影使其自然融入。
- 自包含单文件，Chrome 双击即玩。

### 非目标（YAGNI，明确不做）
- 多关卡、关卡选择
- 变身 / 变大蘑菇等道具系统
- Boss 战
- 射击玩法（角色姿态表中的持枪帧默认不使用）
- 触屏 / 手柄控制
- 存档 / 排行榜
- 标题/通关之外的复杂菜单系统

## 3. 验收标准

1. 在 Chrome 中打开 `super-cat-bro.html` 即可游玩，无需服务器、无需联网。
2. 操作：`← →`/`A D` 移动，`空格`/`↑`/`W` 跳跃，`R` 重开，`Enter` 开始，`M` 静音。
3. 主角能跑、能变高跳；踩小怪头部可消灭小怪并加分回弹；侧面碰小怪扣 1 命；掉进深坑死亡。
4. 吃金币加分；问号块被顶出金币。
5. 到达终点旗杆触发通关结算页（显示总分）。
6. 3 条命用尽出现 Game Over，可回到标题重开。
7. 浏览器控制台无报错。
8. 主角显示的是用户手绘原画切帧（非占位图）。

## 4. 技术架构（方案 A：原生 JS + Canvas，自包含单文件）

单个 `super-cat-bro.html`，内含 `<canvas>`、内联 `<style>`、`<script>`。逻辑按职责切成同文件内的独立模块（各管一摊、可独立理解与测试）：

| 模块 | 职责 | 依赖 |
|------|------|------|
| `Input` | 监听 keydown/keyup，维护按键状态（left/right/jump/restart/start/mute） | — |
| `Player` | 位置、速度、重力、跳跃、AABB 碰撞、状态机（idle/run/jump/dead） | `Input`, `Level` |
| `Level` | 瓦片地图（二维数组）：地面、砖块、问号块、水管、深坑、旗杆；金币与小怪的出生点 | — |
| `Entities` | 金币、小怪（Goomba 式）的 update/draw | `Level`, `Player` |
| `Camera` | 横向跟随主角，钳制到关卡边界 | `Player`, `Level` |
| `Renderer` | 绘制背景（天空/云/山丘）、瓦片、实体、主角 sprite、HUD | 全部 |
| `Audio` | WebAudio 振荡器合成音效 | — |
| `Game` | 固定步长主循环 + 全局状态机（TITLE/PLAYING/DEAD/WIN/GAMEOVER）+ 分数/金币/生命 | 全部 |

- **固定步长游戏循环**：累加器模式，物理以 1/60s（dt = 16.67ms）固定步进，渲染可做插值；保证不同刷新率下手感一致。
- **全局状态机**：`TITLE → PLAYING →（DEAD → 重生 | WIN）`，`DEAD` 且命数为 0 → `GAMEOVER → TITLE`。

## 5. 主角 Sprite 流水线（一次性离线脚本）

脚本 `tools/slice_hero.py`（使用 PIL，环境已确认可用，Pillow 12.2.0）：

1. 打开 `assets/hero_sheet.webp`（2000×2000，2 行 × 3 列，每格约 666×1000）。
2. 切出所需格子。
3. 去白底：近白像素（亮度高于阈值，默认 ~230）转为透明 alpha；阈值边缘做羽化以防白色毛边。
4. 按非透明内容自动裁切（trim 到内容包围盒）。
5. 统一缩放到约 64px 高（≈ 2 个瓦片高），保持纵横比。
6. 导出 PNG 到 `assets/`（保留供检查），并将各帧 base64 内嵌进 `super-cat-bro.html`（保证单文件可分享）。

### 帧 → 游戏状态映射
- `idle` = 格子 (0,0) 正面抱拳
- `run` = 两帧循环：(1,0) 奔跑 + (0,1) 侧面行走，约 10fps
- `jump` = (1,2) 举手欢呼（同时用作通关庆祝姿态）
- 朝左：Canvas 水平翻转（`ctx.scale(-1,1)`），不另存翻转帧
- 不使用：(0,2) 背面、(1,1) 持枪

## 6. 关卡与玩法（经典单关）

### 瓦片与尺寸
- 瓦片 32px。关卡宽约 200 格（一局约 40–60 秒）。
- 瓦片类型：空、地面（草皮顶 + 泥土）、砖块、问号块（撞出金币）、水管（实心障碍）、深坑（无地面）、旗杆（终点）。
- 主角 AABB 约 28px 宽 × 60px 高。

### 手感（默认值，实现期可微调）
- 水平移动：加速度 + 减速度 + 最大速度上限（如 max ≈ 4.5px/帧）。
- 跳跃**变高**：按住跳跃键越久跳得越高（松开提前削减上升速度）。
- coyote-time：离开地面后数帧内仍可起跳（≈ 6 帧）。
- 跳跃缓冲：落地前数帧内按跳，落地瞬间自动起跳（≈ 6 帧）。
- 逐轴 AABB 碰撞解析（先 x 后 y，分别贴合）。

### 实体与规则
- 小怪（Goomba 式）：左右巡逻，遇墙或悬崖边掉头。**踩头**=消灭（+分 + 主角回弹）；**侧面相撞**=扣 1 命。
- 金币：碰到即收集（+分）。
- 问号块：从下方顶撞，顶出 1 枚金币（+分）。
- 深坑：主角落出关卡底部 = 死亡。

### 生命 / 分数 / HUD
- 3 条命。死亡 → 回到关卡起点并扣 1 命。0 命 → Game Over → 回标题。
- 分数来源：金币、踩怪。
- 计时：默认**不做**计时器（可作为后续可选项）。
- HUD：分数、金币数、生命数（屏幕左上角）。

### 终点
- 关卡末尾旗杆，主角到达 → 通关结算页（显示总分）。

## 7. 美术与渲染

- 世界：经典彩色像素风（蓝天、白云、绿色山丘、棕色砖块），整数缩放 + `image-rendering: pixelated`。
- 主角融入处理（解决涂鸦 + 像素混搭的违和感）：
  - 给手绘主角加一圈**白色描边光晕**；
  - 主角脚下加**轻微落地阴影**（椭圆半透明）。
- 配色参考马里奥经典调色板（天空蓝、草绿、泥棕、砖红、金币黄、问号块橙黄）。

## 8. 音效（WebAudio 合成，零素材文件）

用振荡器实时合成，无任何外部音频文件：
- 跳跃：上滑音
- 金币：双音
- 踩怪：闷响
- 死亡：下滑音
- 通关：短琶音

`M` 键静音切换。

## 11. 测试与验收证据

- **纯函数单测（Node）**：把可独立测的逻辑（逐轴 AABB 碰撞解析、跳跃物理、踩踏判定、巡逻掉头）抽成纯函数并断言。
- **浏览器冒烟（Claude Preview MCP）**：本地起静态服务 → 加载页面 → 截图标题页与游戏页 → 确认**控制台无报错** → 脚本模拟按键，确认主角能移动、能踩怪、能到达旗杆。截图与控制台输出作为完成证据，不以"我觉得能跑"代替验证。
- **Sprite 帧质量目检**：切帧后截图各帧，确认去白底无明显毛边、主角显示为真实手绘。

## 12. 风险与缓解

| 风险 | 缓解 |
|------|------|
| 涂鸦主角与像素世界混搭违和 | 白色描边光晕 + 落地阴影；草图观感已经用户确认 |
| 手绘乱线去白底留白色毛边 | 亮度阈值 + 边缘羽化；切帧后截图复核帧质量再定阈值 |
| 跑动可用姿态有限 | 用 idle/侧面/奔跑 三态拼两帧循环 + 代码挤压（squash）补足动感 |
| 单文件过大不易读 | 模块化为同文件内独立对象，文件过大时按职责再拆分 |

## 13. 可调的装饰性参数（已给默认值，用户随时可改）

- 关卡长度（默认约 200 格）
- 命数（默认 3）
- 是否加计时器（默认不加）
- 游戏名称（默认"超级喵拉奥 / Super Cat Bro"）
